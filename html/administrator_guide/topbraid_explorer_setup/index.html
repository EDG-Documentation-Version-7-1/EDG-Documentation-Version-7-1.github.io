TopBraid Explorer Setup

The Explorer server provides users with read-only access to versions of vocabularies or asset collections that have been published from a separate EDG Editor (source) server.

Note

There is no separate module or war file for the explorer server, you’ll use the same edg.war file for both your editor and explorer servers.

Explorer setup

The Explorer server is installed similarly to any EDG server see EDG Installation and Authentication, with the following differences.

1. Explorer installation prerequisites (differences)

  • Verify active database type in EDG. Explorer is required to use the same database type.

  • Explorer should have two security groups, Administrators and Explorer groups. Only these two. Please keep Administrator role to very few users as the Explorer license isn’t enabled for daily use by Administrators. Administrators should be logging into Explorer for configuration changes only. All other users should be in the Customer’s directory group for Explorer role and no other roles present on this Explorer instance.

2. Explorer initial administration (differences)

  • In Server Administration - EDG Rights Management: EDG Permissions Management, assign the ExplorerGroup permission-group to the chosen read only Explorer security role.

  • Configure the Explorer server’s Editor server (URL), user name and password in: Server Administration > Setup – Server Configuration Parameters > Explorer Parameters.

3. Editor (source) EDG administration

For the reverse relationship, configure the Editor-source server’s Explorer server (URL), user name and password in: Server Administration > Setup – Server Configuration Parameters > Explorer Parameters.

4. Publishing to Explorer

After both servers are installed and configured, authorized users can publish edited source asset collections to the Explorer server via the Export tab in EDG collections.
